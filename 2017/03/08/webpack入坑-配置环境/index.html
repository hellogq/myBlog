webpack入坑之配置环境

2017-03-08

什么是webpack？

webpack是近期很火的一款模块加载器兼打包工具，他能把各种资源，例如JS、样式（包含less、sass）图片等都作为模块来使用和处理。
我们可以直接使用requre（XXX）的形式来引入各模块，即使他们可能需要经过编译(比如sass)，但我们无需在上面话费太多心思，因为webpack有着各种健全的加载器(loader)在默默处理这些事情。
近期github上各大主流的(react相关)项目，他们仓库上所展示的示例已经是基于webpack来开发的，比如React-Bootstrap和Redux。

webpack的优势

  1. webpack 是以 commonJS 的形式来书写脚本滴，但对 AMD/CMD 的支持也很全面，方便旧项目进行代码迁移。
  2. 能被模块化的不仅仅是 JS 了。
  3. 开发便捷，能替代部分 grunt/gulp 的工作，比如打包、压缩混淆、图片转base64等。
  4. 扩展性强，插件机制完善，特别是支持 React 热插拔（见 react-hot-loader ）的功能让人眼前一亮。

安装和配置

nodeJS安装，这个是必须滴

如果你作为一名合格的前端人员，想必node环境早就准备的妥妥的了，那么本菇凉就不在这里多废话了，那么，我们来配置webpack的环境。

webpack安装

  • npm install webpack -g 配置全局的webpack
  • 如果常规项目还是把依赖写入package.json包去更人性化
    • npm init
    • npm install webpack --save-dev

运行webpack

直接执行webpack --display-error-details即可，后面的参数“–display-error-details”是推荐加上的，方便出错时能查阅更详尽的信息。
其他主要参数：

  • webpack –config XX.js //使用另一份配置文件来打包
  • webpack –watch //监听变动并自动打包
  • webpack -p //压缩混淆脚本，这个非常重要！
  • webpack -d //生成map映射文件，告知那些模块被最终打包到哪里了

其中的 -p 是很重要的参数，曾经一个未压缩的 700kb 的文件，压缩后直接降到 180kb（主要是样式这块一句就独占一行脚本，导致未压缩脚本变得很大）。
